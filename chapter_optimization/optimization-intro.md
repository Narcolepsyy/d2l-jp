# 最適化と深層学習
:label:`sec_optimization-intro`

この節では、最適化と深層学習の関係、および深層学習における最適化の利用に伴う課題について議論します。
深層学習の問題では、通常まず *損失関数* を定義します。損失関数が得られれば、その損失を最小化しようとして最適化アルゴリズムを用いることができます。
最適化では、損失関数はしばしば最適化問題の *目的関数* と呼ばれます。慣例として、ほとんどの最適化アルゴリズムは *最小化* を扱います。もし目的関数を最大化したい場合は、簡単な解決策があります。目的関数の符号を反転させればよいのです。

## 最適化の目標

最適化は深層学習における損失関数を最小化する手段を与えてくれますが、本質的には、最適化と深層学習の目標は根本的に異なります。
前者は主として目的関数を最小化することに関心があるのに対し、後者は有限個のデータが与えられたときに適切なモデルを見つけることに関心があります。
:numref:`sec_generalization_basics` では、この2つの目標の違いを詳しく議論しました。
たとえば、訓練誤差と汎化誤差は一般に異なります。最適化アルゴリズムの目的関数は通常、訓練データセットに基づく損失関数であるため、最適化の目標は訓練誤差を減らすことです。
しかし、深層学習（より広くは統計的推論）の目標は汎化誤差を減らすことです。
後者を達成するには、最適化アルゴリズムを用いて訓練誤差を減らすことに加えて、過学習にも注意を払う必要があります。

```{.python .input}
#@tab mxnet
%matplotlib inline
from d2l import mxnet as d2l
from mpl_toolkits import mplot3d
from mxnet import np, npx
npx.set_np()
```

```{.python .input}
#@tab pytorch
%matplotlib inline
from d2l import torch as d2l
import numpy as np
from mpl_toolkits import mplot3d
import torch
```

```{.python .input}
#@tab tensorflow
%matplotlib inline
from d2l import tensorflow as d2l
import numpy as np
from mpl_toolkits import mplot3d
import tensorflow as tf
```

上述の異なる目標を説明するために、
経験リスクとリスクを考えてみましょう。
:numref:`subsec_empirical-risk-and-risk` で述べたように、経験リスクは訓練データセット上の平均損失であり、一方リスクはデータ全体の母集団に対する期待損失です。
以下では2つの関数を定義します。
リスク関数 `f`
と経験リスク関数 `g` です。
訓練データが有限個しかないと仮定しましょう。
その結果、ここでの `g` は `f` よりも滑らかではありません。

```{.python .input}
#@tab all
def f(x):
    return x * d2l.cos(np.pi * x)

def g(x):
    return f(x) + 0.2 * d2l.cos(5 * np.pi * x)
```

以下のグラフは、訓練データセット上の経験リスクの最小値が、リスク（汎化誤差）の最小値とは異なる位置にある場合があることを示しています。

```{.python .input}
#@tab all
def annotate(text, xy, xytext):  #@save
    d2l.plt.gca().annotate(text, xy=xy, xytext=xytext,
                           arrowprops=dict(arrowstyle='->'))

x = d2l.arange(0.5, 1.5, 0.01)
d2l.set_figsize((4.5, 2.5))
d2l.plot(x, [f(x), g(x)], 'x', 'risk')
annotate('min of\nempirical risk', (1.0, -1.2), (0.5, -1.1))
annotate('min of risk', (1.1, -1.05), (0.95, -0.5))
```

## 深層学習における最適化の課題

この章では、モデルの汎化誤差ではなく、目的関数を最小化する際の最適化アルゴリズムの性能に特に焦点を当てます。
:numref:`sec_linear_regression` では、最適化問題における解析解と数値解を区別しました。
深層学習では、ほとんどの目的関数は複雑で、解析解を持ちません。その代わりに、数値最適化アルゴリズムを使わなければなりません。
この章で扱う最適化アルゴリズムはすべて、この
カテゴリに属します。

深層学習の最適化には多くの課題があります。特に厄介なのは、局所最小値、鞍点、そして勾配消失です。
それらを見ていきましょう。


### 局所最小値

任意の目的関数 $f(x)$ について、
ある点 $x$ における $f(x)$ の値が、その近傍の他の任意の点における $f(x)$ の値よりも小さいなら、$f(x)$ は局所最小値である可能性があります。
ある点 $x$ における $f(x)$ の値が、定義域全体にわたる目的関数の最小値であるなら、$f(x)$ は大域最小値です。

たとえば、次の関数が与えられたとします。

$$f(x) = x \cdot \textrm{cos}(\pi x) \textrm{ for } -1.0 \leq x \leq 2.0,$$

この関数の局所最小値と大域最小値を近似できます。

```{.python .input}
#@tab all
x = d2l.arange(-1.0, 2.0, 0.01)
d2l.plot(x, [f(x), ], 'x', 'f(x)')
annotate('local minimum', (-0.3, -0.25), (-0.77, -1.0))
annotate('global minimum', (1.1, -0.95), (0.6, 0.8))
```

深層学習モデルの目的関数は通常、多くの局所最適解を持ちます。
最適化問題の数値解が局所最適解の近くにあるとき、最終反復で得られる数値解は、目的関数の勾配が0に近づくか0になるため、目的関数を *大域的に* ではなく *局所的に* しか最小化しないかもしれません。
ある程度のノイズがなければ、パラメータを局所最小値から押し出せないことがあります。実際、これはミニバッチ確率的勾配降下法の有益な性質の1つであり、ミニバッチ間で勾配が自然に変動することで、パラメータを局所最小値から引き離すことができます。


### 鞍点

局所最小値に加えて、鞍点も勾配が消失する別の原因です。*鞍点* とは、関数のすべての勾配が0になるものの、大域最小値でも局所最小値でもない位置のことです。
関数 $f(x) = x^3$ を考えましょう。その1階導関数と2階導関数は $x=0$ で0になります。最適化は、この点で停滞する可能性がありますが、ここは最小値ではありません。

```{.python .input}
#@tab all
x = d2l.arange(-2.0, 2.0, 0.01)
d2l.plot(x, [x**3], 'x', 'f(x)')
annotate('saddle point', (0, -0.2), (-0.52, -5.0))
```

高次元では鞍点はさらに厄介です。以下の例を見てみましょう。関数 $f(x, y) = x^2 - y^2$ を考えます。この関数の鞍点は $(0, 0)$ にあります。これは $y$ に関しては最大値であり、$x$ に関しては最小値です。さらに、実際に *鞍* のように見えます。これが、この数学的性質の名前の由来です。

```{.python .input}
#@tab mxnet
x, y = d2l.meshgrid(
    d2l.linspace(-1.0, 1.0, 101), d2l.linspace(-1.0, 1.0, 101))
z = x**2 - y**2

ax = d2l.plt.figure().add_subplot(111, projection='3d')
ax.plot_wireframe(x.asnumpy(), y.asnumpy(), z.asnumpy(),
                  **{'rstride': 10, 'cstride': 10})
ax.plot([0], [0], [0], 'rx')
ticks = [-1, 0, 1]
d2l.plt.xticks(ticks)
d2l.plt.yticks(ticks)
ax.set_zticks(ticks)
d2l.plt.xlabel('x')
d2l.plt.ylabel('y');
```

```{.python .input}
#@tab pytorch, tensorflow
x, y = d2l.meshgrid(
    d2l.linspace(-1.0, 1.0, 101), d2l.linspace(-1.0, 1.0, 101))
z = x**2 - y**2

ax = d2l.plt.figure().add_subplot(111, projection='3d')
ax.plot_wireframe(x, y, z, **{'rstride': 10, 'cstride': 10})
ax.plot([0], [0], [0], 'rx')
ticks = [-1, 0, 1]
d2l.plt.xticks(ticks)
d2l.plt.yticks(ticks)
ax.set_zticks(ticks)
d2l.plt.xlabel('x')
d2l.plt.ylabel('y');
```

関数の入力が $k$ 次元ベクトルで出力がスカラーであると仮定すると、そのヘッセ行列は $k$ 個の固有値を持ちます。
関数の解は、関数の勾配が0である位置において、局所最小値、局所最大値、または鞍点になりえます。

* 関数のヘッセ行列の固有値が、勾配が0の位置で全て正なら、その関数は局所最小値を持ちます。
* 関数のヘッセ行列の固有値が、勾配が0の位置で全て負なら、その関数は局所最大値を持ちます。
* 関数のヘッセ行列の固有値が、勾配が0の位置で負と正の両方を含むなら、その関数は鞍点を持ちます。

高次元問題では、少なくとも *いくつか* の固有値が負である確率はかなり高くなります。そのため、鞍点は局所最小値よりも起こりやすくなります。次の節で凸性を導入するときに、この状況に対するいくつかの例外を議論します。要するに、凸関数とはヘッセ行列の固有値が決して負にならない関数です。残念ながら、深層学習の問題の大半はこのカテゴリに入りません。それでも、最適化アルゴリズムを学ぶうえで非常に有用な道具です。

### 勾配消失

おそらく最も厄介な問題は勾配消失です。
:numref:`subsec_activation-functions` で扱った、よく使われる活性化関数とその導関数を思い出してください。
たとえば、関数 $f(x) = \tanh(x)$ を最小化したいとして、たまたま $x = 4$ から始めたとします。見てのとおり、$f$ の勾配はほとんど0です。
より具体的には、$f'(x) = 1 - \tanh^2(x)$ なので、$f'(4) = 0.0013$ です。
その結果、進展が見られるまで最適化は長い間停滞します。これは、ReLU活性化関数が導入される以前に深層学習モデルの訓練がかなり難しかった理由の1つです。

```{.python .input}
#@tab all
x = d2l.arange(-2.0, 5.0, 0.01)
d2l.plot(x, [d2l.tanh(x)], 'x', 'f(x)')
annotate('vanishing gradient', (4, 1), (2, 0.0))
```

見てきたように、深層学習の最適化には課題が山積しています。幸いなことに、性能が良く、初心者でも使いやすい堅牢なアルゴリズムが幅広く存在します。さらに、必ずしも *唯一の* 最良解を見つける必要はありません。局所最適解や、その近似解であっても、依然として非常に有用です。

## まとめ

* 訓練誤差を最小化しても、汎化誤差を最小化する最良のパラメータ集合が見つかるとは限りません。
* 最適化問題には多くの局所最小値が存在しうる。
* 一般に問題は凸ではないため、さらに多くの鞍点が存在しうる。
* 勾配消失は最適化を停滞させることがあります。しばしば、問題の再パラメータ化が役立ちます。パラメータの良い初期化も有益です。
